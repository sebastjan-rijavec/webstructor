# Worklog ‚Äî 2026-05-14

Single-day development log for Webstructor. All changes sit on top of the
initial commit (`feee481`) and ship as one follow-up commit.

## Headline

- Viewport gained image-based lighting, named axis views (3D / front / back /
  left / right / top / bottom), animated transitions, and animated FOV.
- Editing gained true multi-select transforms via a centroid pivot.
- Selection now renders as box outlines instead of material tinting.
- Library gained GLB-backed assets (Pony, Star Destroyer) with caching.
- UI switched from a dark to a light theme.

## Changes by area

### Library ‚Äî GLB assets

- New `src/library/models.ts`: GLTFLoader-backed factories with a per-URL
  promise cache; first instantiation triggers the network/load, subsequent
  ones reuse the cached `Group` and deep-clone geometry + material per
  instance so per-object edits stay isolated.
- `registry.ts`: `create()` may now return `Object3D | Promise<Object3D>`;
  `instantiate()` is `async`. Existing primitive factories are unchanged
  (sync factories still satisfy the union).
- `index.ts`: side-effect import of `./models` registers the GLB entries
  next to the primitives.
- `data/`: ships `pony_cartoon.glb` and
  `star_wars_imperial_ii_star_destroyer.glb`. Imported via Vite's `?url`
  suffix so production builds resolve hashed asset URLs.

### Scene ‚Äî viewport rewrite

`src/scene/viewport.ts` grew from a small bootstrap into the camera/animation
hub of the app:

- **IBL**: `PMREMGenerator` prefilters a `RoomEnvironment` once at startup
  and assigns it to `scene.environment`. Direct lights toned down (key 1.1,
  fill 0.25, no hemisphere) since IBL now provides ambient diffuse.
- **Named views**: `ViewName = perspective | front | back | left | right | top | bottom`. Axis views lock orbit rotation (pan + zoom remain) and frame the scene against the bbox's *near face* ‚Äî not its center ‚Äî to compensate for perspective foreshortening.
- **Animated transitions** (`setView`): slerp-based arc interpolation of the
  camera direction vector around the orbit target, with cubic ease-in/out.
  Antipodal axis pairs (front‚Üîback, left‚Üîright, top‚Üîbottom, etc.) route
  through a great-circle waypoint table so the path stays on a single arc.
- **Two-phase framing**: a 3D‚Üîaxis transition with a selection bbox runs as
  (1) rotate to the target angle while framed on the whole scene, then
  (2) dolly into the selection. Each phase eases independently so the camera
  comes to rest at the midpoint. Axis‚Üîaxis transitions skip the split.
- **FOV animation** (`setFov`): independent eased animation; interrupting an
  in-flight FOV change snaps to its target before starting the new one.
- **`frame(bbox)`**: dollies along the current view direction to fit a
  bbox. F key + Frame toolbar button call this on the current selection
  (falls back to all content).
- **`onTick`**: per-frame subscription used by the main module to refresh
  selection box helpers.

### Editing ‚Äî multi-select transforms

`src/editing/transform.ts`:

- Replaced `attach(obj)` with `setObjects(objs[])`.
- 0 selected ‚Üí gizmo detached. 1 selected ‚Üí gizmo attaches directly (uses
  the object's local axes). >1 selected ‚Üí gizmo attaches to an invisible
  `__multiPivot` placed at the selection centroid; on drag, the
  world-space delta (`pivot.matrixWorld * pivotStartInv`) is applied to
  each selected object's stored start matrix and decomposed back into
  local TRS through the parent's inverse.
- `DragState` discriminates single vs. multi drags; both produce a
  `TransformRecord[]` on commit.
- After a multi-drag ends, the pivot is re-centered with identity
  rotation/scale so the next drag starts cleanly.

`src/editing/commands.ts`:

- `transformCommand` now takes `TransformRecord[]` and applies/undoes the
  whole batch atomically. Label adapts (`Transform N objects` vs.
  `Transform Box`).
- `duplicateCommand` deep-clones geometry and material, so editing a
  duplicate no longer bleeds into its source (matches the per-instance
  isolation that the GLB factory already enforces).

### Editing ‚Äî selection outlines

`src/editing/selection.ts`:

- Replaced material `emissive` tinting with `THREE.BoxHelper` instances
  parented under `viewport.helpers` (so they're excluded from GLB export).
- Helpers tagged with `userData.__selectionHelper = true` and refreshed
  per-frame via `updateHelpers()`, called from `viewport.onTick`.
- Restoring original emissive colors is no longer needed, which removes
  the WeakMap of original values and works for non-StandardMaterial
  meshes loaded from GLBs.
